    How to Setup AI Chat Assistant¶

    Configure the Unity Mathematics AI Assistant¶

    This guide walks you through setting up the AI-powered chat assistant that can answer questions about unity mathematics, run proofs, and generate visualizations.

    Prerequisites¶

    • Python 3.8+ installed
    • OpenAI API key (for GPT-powered responses)
    • Git repository cloned locally

    Quick Setup¶

    1. Install Dependencies¶

    # Navigate to AI agent directory
cd ai_agent

# Install required packages
pip install -r requirements.txt

    Key dependencies installed: - openai - GPT API integration - faiss-cpu - Vector search for code embeddings - sentence-transformers - Text embedding generation - fastapi - Backend API server - streamlit - Optional web interface

    2. Configure API Keys¶

    Create environment file:

    # Copy example environment file
cp .env.example .env

# Edit .env file
nano .env  # or your preferred editor

    Add your API keys: 

    # Required: OpenAI API key
OPENAI_API_KEY="sk-proj-your-actual-key-here"

# Optional: For enhanced features
ANTHROPIC_API_KEY="your-anthropic-key"
HUGGINGFACE_API_TOKEN="your-huggingface-token"

# Server configuration
HOST=localhost
PORT=8000
DEBUG=true

    3. Build Knowledge Index¶

    Generate embeddings for the entire repository:

    # Run the indexing script
python prepare_index.py

    What this does: - Scans all Python, R, and Markdown files - Generates semantic embeddings for code and documentation - Creates searchable vector database - Builds context-aware response system

    Expected output: 

    🔍 Scanning repository files...
📁 Found 245 files to process
🧠 Generating embeddings...
💾 Saving vector index...
✅ Knowledge base created successfully!

    4. Start the AI Backend¶

    Launch the API server:

    python app.py

    Server startup output: 

    🤖 Een Unity AI Assistant Starting...
📚 Loading knowledge base...
🌐 Starting server on http://localhost:8000
✅ AI Assistant ready for unity mathematics queries!

    Web Interface Setup¶

    5. Launch Web Chat¶

    In parallel terminal, start the web interface:

    # Navigate to website directory
cd ../website

# Start local web server
python -m http.server 8080

    Access the interface: - Open browser to http://localhost:8080 - Click on "AI Chat" or navigate to chat interface - Start asking questions about unity mathematics!

    6. Test the AI Assistant¶

    Try these example queries:

    Basic Unity Questions: 

    "Explain how 1+1=1 in phi-harmonic mathematics"
"Show me a quantum unity proof"
"What is the Omega equation?"

    Code Queries: 

    "How do I run a unity proof?"
"Show me the consciousness field equation"
"Generate quantum unity visualization code"

    Interactive Requests: 

    "Run a proof that 1+1=1"
"Create a visualization of phi-harmonic scaling"
"Generate a consciousness field animation"

    Advanced Configuration¶

    7. Customize AI Responses¶

    Edit the AI configuration:

    # In ai_agent/config.py
AI_CONFIG = {
    "model": "gpt-4",  # or "gpt-3.5-turbo"
    "temperature": 0.7,
    "max_tokens": 2000,
    "unity_focus": True,  # Emphasize unity mathematics
    "phi_harmonic_mode": True,  # Use φ-harmonic reasoning
    "consciousness_integration": True  # Include consciousness context
}

    8. Add Custom Knowledge Sources¶

    Extend the knowledge base:

    # In ai_agent/knowledge_sources.py
CUSTOM_SOURCES = {
    "unity_axioms": "docs/theory/",
    "proof_methods": "proofs/",
    "visualization_guides": "viz/",
    "consciousness_models": "core/consciousness.py"
}

# Rebuild index with custom sources
python prepare_index.py --include-custom

    9. API Endpoints¶

    The AI backend provides these endpoints:

    Chat Endpoint: 

    curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Explain unity mathematics"}'

    Code Search: 

    curl -X POST http://localhost:8000/search-code \
  -H "Content-Type: application/json" \
  -d '{"query": "quantum unity proof"}'

    Generate Proof: 

    curl -X POST http://localhost:8000/generate-proof \
  -H "Content-Type: application/json" \
  -d '{"framework": "quantum"}'

    Docker Deployment¶

    11. Containerized Setup¶

    For production deployment:

    # Dockerfile for AI assistant
FROM python:3.11-slim

WORKDIR /app

# Copy requirements and install dependencies
COPY ai_agent/requirements.txt .
RUN pip install -r requirements.txt

# Copy application code
COPY ai_agent/ .
COPY core/ ./core/
COPY proofs/ ./proofs/

# Build knowledge index
RUN python prepare_index.py

# Expose port
EXPOSE 8000

# Start server
CMD ["python", "app.py"]

    Build and run: 

    docker build -t een-ai-assistant .
docker run -p 8000:8000 -e OPENAI_API_KEY=$OPENAI_API_KEY een-ai-assistant

    Troubleshooting¶

    Common Issues¶

    Issue: API key not recognized 

    # Solution: Verify environment variable
echo $OPENAI_API_KEY
# Should show your key (partially masked)

    Issue: Knowledge base not loading 

    # Solution: Rebuild index
rm -rf ai_agent/knowledge_base/
python ai_agent/prepare_index.py

    Issue: Server connection errors 

    # Solution: Check port availability
netstat -an | grep 8000
# If occupied, use different port

    Issue: Slow response times 

    # Solution: Optimize embedding search
# In ai_agent/config.py
SEARCH_CONFIG = {
    "max_results": 5,  # Reduce from default 10
    "similarity_threshold": 0.8,  # Increase threshold
    "use_gpu": True  # If available
}

    14. Integration Testing¶

    Test the AI assistant:

    # test_ai_assistant.py
import requests
import json

def test_unity_question():
    response = requests.post("http://localhost:8000/chat", 
                           json={"message": "Prove 1+1=1"})
    assert response.status_code == 200
    data = response.json()
    assert "unity" in data["response"].lower()
    print("✅ Unity question test passed")

def test_code_search():
    response = requests.post("http://localhost:8000/search-code",
                           json={"query": "unity_add"})
    assert response.status_code == 200
    data = response.json()
    assert len(data["results"]) > 0
    print("✅ Code search test passed")

# Run tests
test_unity_question()
test_code_search()
print("🎉 All AI assistant tests passed!")

    Performance Optimization¶

    15. Caching and Speed¶

    Improve response times:

    # In ai_agent/app.py
from functools import lru_cache
import redis

# Redis caching for frequent queries
redis_client = redis.Redis(host='localhost', port=6379, db=0)

@lru_cache(maxsize=1000)
def cached_embedding_search(query):
    """Cache embedding search results"""
    return search_knowledge_base(query)

def cache_response(query, response):
    """Cache AI responses for 1 hour"""
    redis_client.setex(f"ai_response:{query}", 3600, response)
